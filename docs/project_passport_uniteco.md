# Unit-Economy Growth Simulator
*(Мобильный интерактивный стенд для моделирования продуктовых метрик и подготовки к техническим интервью)*

## 1. Концепция продукта (Product Essence)

**Unit-Economy Growth Simulator** — это мобильный интерактивный дашборд и симулятор бизнес-сценариев, развернутый локально на смартфоне в среде Termux. Продукт спроектирован как универсальный стенд абстрактного бизнеса для изучения, расчетов и визуализации юнит-экономики продукта.

Симулятор решает проблему отсутствия практического понимания продуктовых метрик «на кончиках пальцев» и полностью заменяет громоздкие Excel-таблицы на смартфоне. 

**Ключевые характеристики решения:**
* **Synergy of Analytics & Engineering:** Демонстрация продуктового мышления (Product Sense) через самостоятельную разработку аналитического инструмента.
* **100% Mobility:** Полностью локальный запуск в эмуляторе терминала (Termux) с доступом через мобильный браузер.
* **Educational Focus:** Интегрированный AI-модуль тестирования для проверки усвоенных знаний и закрепления связей между метриками.
* **Target Audience:** Первично — автор продукта (обучение, сохранение стрика), вторично — рекрутеры и нанимающие менеджеры (как мощное интерактивное демо хард-скиллов), с потенциалом масштабирования в open-source.

## 2. Ключевой функционал (Core Features)

Продукт охватывает полный жизненный цикл юнит-экономики, от привлечения до удержания:

* **Full-Cycle Calculator:** Последовательный расчет по цепочке: Воронка -> Стоимость привлечения (CAC) -> Удержание (Retention) -> Пожизненная ценность (LTV) -> Возврат инвестиций (ROMI). Включает поддержку кастомных формул.
* **Сценарное моделирование ("Что если?"):** Гибкая система оценки изменений. Включает реактивный UI (перестроение графиков в реальном времени при движении ползунков), сохранение пресетов для визуального сравнения и автоматическую генерацию текстовых отчетов о влиянии изменений на метрики.
* **Когортный анализ:** Построение и визуализация полноценной треугольной матрицы удержания (Retention Matrix) для анализа "старения" когорт.
* **Мультиформатный ввод данных:** Комбинация мобильных ползунков (sliders), полей точного ввода и возможности загрузки сырых CSV-файлов.
* **AI-Генератор тестов (Self-Check Module):** Интеграция с OpenRouter API для автоматической генерации тестовых вопросов по внедренным в проект метрикам и терминам.

## 3. Проблематика и операционные барьеры (The Pain)

Инструмент имитирует и решает классические «боли» продуктового аналитика и бизнеса:

* **Информационная фрагментация:** Сложность сведения разрозненных маркетинговых и продуктовых данных в единую картину прибыльности.
* **Низкая скорость принятия решений:** Необходимость быстро давать бизнесу математически обоснованный ответ на вопросы вида "а что если изменим цену или увеличим бюджет?".
* **Поиск Bottlenecks:** Трудности в выявлении «узких горлышек» в воронке конверсии.
* **Когнитивные искажения бизнеса:** Симулятор помогает избегать классических ловушек — иллюзии, что рост базы всегда означает рост прибыли при минусовой экономике; непонимания влияния оттока (Churn) на долгосрочную выручку; слепоты к кассовым разрывам (Payback Period).

* **Test Accuracy Rate (%):** Процент правильных ответов в интегрированном модуле AI-тестирования. Эта метрика отражает реальное усвоение материала автором продукта, а не просто факт написания кода.

### Операционные и карьерные метрики (Secondary Metrics)
* **Definition of Done (MVP):** Работает минимум 5 ползунков метрик, графики перестраиваются без перезагрузки страницы (Reactive UI).
* **Technical Stability:** Код откомментирован, математика покрыта Unit-тестами и сверена с эталонным Excel, приложение запускается в Termux без ошибок.
* **Career Impact:** Успешная презентация проекта на интервью и получение оффера. Проект оформлен на GitHub с Readme и архитектурным описанием.

## 7. Архитектура системы (Technical Pipeline & Architecture)

### Мобильная адаптивность (UI Layer)
Реализован **Реактивный Dashboard** (NiceGUI + Tailwind CSS).
* Верхняя часть экрана: фиксированный хедер (Fixed Header) с ключевыми метриками-индикаторами (LTV, CAC, ROMI), которые всегда перед глазами.
* Нижняя часть: скроллящаяся область с блоками параметров (маркетинг, продукт) и визуализацией (Plotly).

### Структура кода (Modular Architecture)
Строгое разделение логики по папкам:
* `/core` — математическое ядро и расчеты (DuckDB, Pandas).
* `/ui` — веб-компоненты и реактивность (NiceGUI).
* `/db` — хранилище и схемы (SQLite, Pydantic).

### Парадигма данных: Гибридный путь
Архитектура имитирует взрослые корпоративные системы (OLTP + OLAP), разделяя ответственность:
1. **Хранение намерений (SQLite):** Сохранение состояний ползунков, конфигураций сценариев и автосохранений.
2. **Валидация (Pydantic):** Жесткий контроль типов и форматов данных перед их обработкой, защита от ошибок ввода.
3. **Аналитические расчеты (DuckDB + NumPy/Pandas):** Оконные функции, агрегации и расчет когорт. DuckDB извлекает данные и выполняет тяжелые SQL-операции в оперативной памяти на максимальной скорости.

**Структура доменных таблиц БД:**
1. `Marketing_Params` — бюджеты, стоимость кликов, конверсии воронки.
2. `Product_Params` — метрики удержания (Retention) и оттока (Churn).
3. `Economy_Results` — итоговые расчеты LTV, CAC, прибыли и окупаемости.

## 8. Технологический стек (Technical Stack)

**Язык и Среда:**
* Python 3.10+
* Termux (Эмулятор Linux для Android) — среда локального запуска в отпуске.

**Backend & Data Processing:**
* **DuckDB** — аналитическое ядро SQL для сложных расчетов.
* **Pandas / NumPy** — векторизованные вычисления и работа с матрицами (когортами).
* **Pydantic** — строгая валидация моделей данных.
* **SQLite** — легковесное постоянное хранилище конфигов.
* **HTTPX** — асинхронный клиент для взаимодействия с внешними API (Google Sheets, OpenRouter).

**Frontend & Data Visualization:**
* **NiceGUI** — Python-фреймворк для реактивного Web-UI.
* **Plotly** — интерактивные графики, адаптируемые под мобильные экраны.

## 9. Интеграция в экосистему портфолио (Career Alignment)

Проект **Unit-Economy Growth Simulator** не существует в вакууме. Он является третьим, завершающим пазлом в формировании единой экосистемы компетенций для позиции Junior Product Analyst:

1. **Autonomous Stock Shadow (ASSE):** Работа с "грязными" данными, ETL-пайплайны, парсинг, автоматизация бизнеса.
2. **Personal Cognitive Analytics (PCAR):** Работа с AI (Inference), векторными базами данных, RAG, анализ текстов.
3. **UnitEco Simulator:** Переход от технического слоя к бизнес-слою. Глубокое понимание метрик, денег и влияния продукта на прибыль.

Стек всех трех проектов унифицирован (Python, БД, полная автономность), что демонстрирует системный подход кандидата к созданию законченных IT-решений.
